# Гайд по деплою: фронт на Vercel + сервер на Render

Ниже — полный рабочий сценарий для этого проекта:
- `frontend` (React/CRACO) деплоится на **Vercel**;
- `backend` (Express) деплоится на **Render**.

## 1) Подготовка репозитория

1. Убедись, что код залит в GitHub/GitLab/Bitbucket.
2. Проверь, что в проекте есть две папки:
   - `frontend`
   - `backend`

## 2) Деплой backend на Render

### 2.1 Создание сервиса

1. Открой [https://render.com](https://render.com) и зайди в аккаунт.
2. Нажми **New +** → **Web Service**.
3. Подключи репозиторий с проектом.
4. В настройках сервиса укажи:
   - **Root Directory**: `backend`
   - **Runtime**: `Node`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`

### 2.2 Переменные окружения backend

Добавь в Render → Service → **Environment**:

Обязательные:
- `PORT` — не задавать вручную (Render сам передаст), либо оставь пустым.
- `ADMIN_TOKEN` — токен для админки (любой надежный секрет).

Опциональные (если нужны уведомления в Telegram):
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`

### 2.3 Доступ к файловым загрузкам

Сервер хранит файлы в `backend/uploads`. На бесплатных/обычных инстансах Render файловая система **эфемерная** — файлы могут исчезать после перезапуска.

Если важно постоянное хранение изображений, лучше использовать внешний storage (S3/Cloudinary). Без этого после перезапуска часть загруженных файлов может пропасть.

### 2.4 Проверка backend

После деплоя получишь URL вида:
`https://<your-backend>.onrender.com`

Проверь, что сервер отвечает:
- открой URL в браузере;
- или проверь API-эндпоинт, который уже есть в проекте.

## 3) Деплой frontend на Vercel

### 3.1 Создание проекта

1. Открой [https://vercel.com](https://vercel.com).
2. Нажми **Add New…** → **Project**.
3. Импортируй тот же репозиторий.
4. В настройках проекта укажи:
   - **Root Directory**: `frontend`
   - **Framework Preset**: `Create React App` (или Auto Detect)
   - **Build Command**: `npm run build`
   - **Output Directory**: `build`

### 3.2 Переменные окружения frontend

Добавь в Vercel → Project → **Settings** → **Environment Variables**:

- `REACT_APP_BACKEND_URL=https://<your-backend>.onrender.com`

Важно: без `REACT_APP_BACKEND_URL` фронт будет ходить на локальный `http://localhost:8001`.

### 3.3 Redeploy

После добавления переменных нажми **Redeploy** (или сделай новый коммит), чтобы фронт пересобрался с корректным API URL.

## 4) CORS и домены

Когда фронт на Vercel начнет обращаться к backend на Render, нужен корректный CORS на сервере.

Если увидишь ошибку CORS в браузере:
1. Открой backend-конфиг CORS.
2. Добавь туда домен фронта с Vercel (например, `https://my-app.vercel.app`).
3. Перезадеплой backend на Render.

## 5) Минимальный чек-лист после деплоя

1. Открывается frontend URL от Vercel.
2. Список/данные на фронте подтягиваются с Render API.
3. Работают формы, создание/редактирование в админке.
4. Изображения загружаются и отображаются.
5. Нет ошибок CORS в DevTools → Console/Network.

## 6) Обновления проекта

После каждого `git push`:
- Render пересоберет backend (если включен Auto Deploy);
- Vercel пересоберет frontend.

Если менял env-переменные — делай ручной redeploy для каждого сервиса.

## 7) Частые проблемы и быстрые решения

### Проблема: фронт не видит API
- Проверь `REACT_APP_BACKEND_URL` в Vercel.
- Убедись, что URL начинается с `https://` и без лишнего `/` в конце.
- Пересобери frontend.

### Проблема: 401/403 в админке
- Проверь `ADMIN_TOKEN` на Render.
- Убедись, что frontend отправляет тот же токен.

### Проблема: пропадают загруженные файлы
- Это ожидаемо для эфемерного диска на Render.
- Перенеси хранение файлов во внешний storage.

### Проблема: долго «просыпается» backend
- На free-тарифах Render сервис может засыпать.
- Первый запрос после сна может быть медленным.

## 8) Рекомендуемая прод-конфигурация

- Отдельные окружения: `staging` и `production`.
- Разные `ADMIN_TOKEN` для каждого окружения.
- Логи и алерты (Render Logs + внешние мониторинги).
- Внешнее хранилище изображений (Cloudinary/S3).
